# Bulletin Board Backend

Backend-часть приложения для размещения объявлений.

Проект реализует REST API для работы с пользователями, объявлениями, комментариями и изображениями.

---

# Функциональность

- регистрация пользователя
- авторизация пользователя
- получение профиля пользователя
- обновление профиля
- смена пароля
- просмотр объявлений без авторизации
- создание объявления
- редактирование объявления
- удаление объявления
- загрузка изображения объявления
- добавление комментариев
- редактирование комментариев
- удаление комментариев
- загрузка аватарки пользователя

---

# Используемые технологии

- Java 17
- Spring Boot
- Spring Web
- Spring Security
- Spring Data JPA
- PostgreSQL
- Hibernate
- Lombok
- Swagger / OpenAPI
- Docker

---

# Архитектура проекта

Проект реализован по классической многослойной архитектуре:

Controller → Service → Repository → Database

Controller — принимает HTTP-запросы  
Service — содержит бизнес-логику  
Repository — взаимодействует с базой данных  
Mapper — преобразует Entity в DTO и обратно  
Entity — модели базы данных  
DTO — объекты передачи данных

---

# Работа с изображениями

Изображения сохраняются в файловой системе.

В базе данных хранится только путь к изображению.

Получение изображения выполняется через endpoint:

GET /images/{fileName}

---

# Безопасность

Используется Spring Security и Basic Authentication.

Пользователи загружаются из базы данных через кастомный сервис MyUserDetailsService.

Реализована ролевая модель:

USER  
ADMIN

Обычный пользователь может:
- редактировать свои объявления
- удалять свои объявления
- редактировать свои комментарии

Администратор может:
- редактировать любые объявления
- удалять любые объявления
- редактировать любые комментарии

---

# База данных

Используется PostgreSQL.

Основные сущности:

UserEntity  
AdEntity  
CommentEntity

Связи:

- пользователь может иметь несколько объявлений
- пользователь может иметь несколько комментариев
- объявление может иметь несколько комментариев

---

# Запуск проекта

1. Запустить PostgreSQL
2. Настроить application.properties
3. Запустить приложение

mvn spring-boot:run

Swagger доступен по адресу:

http://localhost:8080/swagger-ui/index.html
